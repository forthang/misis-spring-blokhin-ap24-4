
# Лабораторная работа 1

## Описание

В данной лабораторной работе были освоены базовые операции с Git, такие как клонирование, добавление, коммит, ветвление, слияние и разрешение конфликтов. Также была создана система сборки для C++ проектов с использованием CMake, что позволило понять процесс сборки многофайлового проекта.



## Шаги по сборке проекта

1. **Создание репозитория:**
   - Создан новый репозиторий на GitHub для лабораторной работы.
   - Репозиторий был клонирован на локальный компьютер.

2. **Настройка CMake:**
   - В корне репозитория была создана директория `src`.
   - В директории `src` были созданы два файла: `main.cpp` и `utils.cpp`.
     - `main.cpp` содержит функцию `main`, которая использует функции из `utils.cpp`.
     - `utils.cpp` содержит несколько простых функций (например, сложение и вычитание).
   - В корне репозитория был создан файл `CMakeLists.txt`, который собирает проект, включая `main.cpp` и `utils.cpp`.
   - Проект был успешно собран с помощью следующих команд:
     ```
     mkdir build
     cd build
     cmake ..
     make
     ```

3. **Работа с Git:**
   - Инициализирован Git репозиторий в корне проекта (если не был инициализирован автоматически).
   - Все файлы были добавлены в репозиторий и сделан первый коммит с сообщением:
     ```
     Initial commit: CMake project setup
     ```
   - Создана новая ветка с именем `feature-utils`.
   - В ветке `feature-utils` в `utils.cpp` была добавлена функция умножения, а также обновлен `main.cpp` для использования этой функции.
   - Закоммичены изменения в ветке `feature-utils` с сообщением:
     ```
     feat: Add multiplication function to utils
     ```
   - Переключение обратно на основную ветку ( `main`).
   - В основной ветке внесены изменения в `main.cpp` ( изменен вывод программы) и закоммичены с сообщением:
     ```
     fix: Update main output message in main branch
     ```
   - Попытка слить ветку `feature-utils` в основную ветку:
     ```
     git merge feature-utils
     ```
   - Возник конфликт слияния, он был разрешен путем выбора нужных изменений и закоммичен результат слияния.

4. **Отправка изменений:**
   - Все ветки и коммиты были отправлены в удаленный репозиторий:
     ```
     git push --all origin
     ```
